  🎮
IFC.js ist eine JavaScript-Bibliothek zum Laden, Anzeigen und Bearbeiten von IFC-Modellen im Browser. Testen Sie die [Live-Demo](https://ifcjs.github.io/web-ifc-viewer/examples/simple-html/) mit Ihrem IFC-Modell und entdecken Sie die Leistungsfähigkeit von IFC.js.

## Was ist diese Bibliothek?

### Was ist IFC?

  🏠
Architekten zeichnen Gebäudepläne nicht mehr einzeln, sondern erstellen 3D-Modelle von Gebäuden, ähnlich wie die Häuser in [The SIMS](https://www.ea.com/es-es/games/the-sims) modelliert werden. Diese Arbeitsweise wird BIM (Building Information Model) genannt, weil die erstellten Modelle sowohl Geometrie als auch Informationen enthalten.

  🔥
Ein Gebäude ist jedoch so komplex, dass es keine einzelne Anwendung gibt, die diese Modelle in ihrer Gesamtheit erstellen kann. Jede Anwendung hat eine bestimmte Aufgabe: Geometrie modellieren, Strukturen berechnen, Energiesimulationen durchführen, Projektdokumente erstellen und so weiter.

  👀
Diese Tools kommen von mehreren Entwicklern auf der ganzen Welt und jedes arbeitet in seinem eigenen Format. Mit anderen Worten: Das von einem Statikprogramm erstellte Strukturmodell kann nicht von einer Energiesimulationssoftware gelesen werden und umgekehrt. Wie ist es also möglich, ein Gebäudemodell aus mehreren Tools zu erstellen?

  🎉
Die Antwort ist [**IFC**](https://www.buildingsmart.org/standards/bsi-standards/industry-foundation-classes/), ein von BuildingSMART erstelltes Format, das diese 3D-Modelle von Gebäuden enthält. Es ist offen, also kann es jeder lesen und schreiben. Juhu!

  <IfcAlert>
Dank IFC kann jeder Entwickler frei BIM-Apps erstellen, die mit den Apps der großen Entwicklungsfirmen im Bausektor kommunizieren können.
  </IfcAlert>


### Das Problem der IFC

Jeder, der eine BIM-Anwendung machen möchte, **muss IFC-Dateien lesen und schreiben**, um Informationen aus anderen Tools importieren und exportieren zu können. Leider ist IFC ein Format, das schwer zu lesen und zu schreiben ist. Es gibt Tausende von Seiten an Dokumentation und die Erstellung und Pflege eines selbst erstellten IFC-Parsers ist eine riesige Aufgabe.

  <IfcCard title="
Können Sie sich IFC leisten?"
  >
    💸
Das ist etwas, was sich nur Firmen leisten können, die mehrere Entwickler haben, die in Vollzeit daran arbeiten. So offen klingt das gar nicht mehr, oder?
</IfcCard>

Falls Sie neugierig sind, wie eine IFC im Inneren aussieht, haben Sie hier ein Beispiel. Stellen Sie sich vor, Hunderttausende von Zeilen wie diese zu parsen:

  ```js
  #6699= IFCCARTESIANPOINT((0.,-1.7053025E-13));
  #6701= IFCAXIS2PLACEMENT2D(#6699,#23);
  #6703= IFCCARTESIANPOINT((892.,-253.399999,150.));
  #6705= IFCAXIS2PLACEMENT3D(#6703,#15,#19);
  #6706= IFCEXTRUDEDAREASOLID(#6702,#6705,#19,506.8);
  #6707= IFCCOLOURRGB($,0.50196,0.501913,0.501960);
  ```

  💀
Selbst wenn Sie bereit sind, einen Parser von Grund auf zu erstellen und ihn bei jeder Änderung des IFC-Schemas zu pflegen, gibt es mehrere nicht-triviale Fragen, um dieses Problem anzugehen. Wie verwaltet man den Speicher in sehr großen Dateien? Wie lässt sich die Geometrieerzeugung effizient implementieren? Was ist zu tun, wenn eine IFC nicht korrekt definiert wurde?

### IFC.js zur Rettung

Es wäre nicht sinnvoll, wenn alle Entwickler einen eigenen IFC-Dateireader und -Writer für ihre Anwendungen implementieren müssten. Vor allem, wenn wir alle das Gleiche wollen: **Importieren und Exportieren von Geometrie und Daten**.

  <IfcCard>
IFC.js ist eine JavaScript-Bibliothek, die das <b>Lesen und Schreiben von IFC-Dateien</b> super einfach macht.
  </IfcCard>

Dies ermöglicht es Entwicklern von Architektur- und Bauanwendungen, mühelos in IFC zu arbeiten und sich auf die Funktionalitäten zu konzentrieren, die wirklich einen Mehrwert für ihr Unternehmen darstellen.

## IFC.js kurz und bündig

IFC.js hat zwei Schwerpunkte:

    icon="💎"

  </IfcCard>

  <IfcCard

Das Erstellen einer BIM-Anwendung mit IFC.js ist so einfach wie das Erstellen einer Webseite mit JavaScript, HTML und CSS. Wenn Sie keine IFC-Datei zur Verfügung haben, versuchen Sie, eine von [diesen](https://drive.google.com/drive/folders/1MwRsL3ZnlyhWEVUko9wAnklaMne3pp3h?usp=sharing) zu laden.

  ```js
  import { IfcLoader } from 'web-ifc-three';
  import { Scene } from 'three';

  //Erzeugt THREE.js-Szene
  const scene = new Scene();

  //...

  //Lädt die IFC und fügt sie der Szene hinzu
  const ifcLoader = new IfcLoader();
  ifcLoader.load(ifcURL, (geometry) => scene.add(geometry));
  ```

## Für wen ist diese Bibliothek

Diese Bibliothek ist für jeden gedacht, der BIM-Anwendungen entwickeln möchte. Dazu gehören sowohl Entwickler, die Anwendungen für die Bauindustrie erstellen, als auch Architekten und andere Fachleute, die die Daten in ihren IFC-Dateien nutzen möchten.

Zur Verwendung dieser Bibliothek sind Grundkenntnisse in JavaScript, HTML und CSS erforderlich. Diese Kenntnisse gehen über den Rahmen dieser Dokumentation hinaus. Außerdem müssen Sie eine 3D-Bibliothek verwenden, um die Geometrie der IFC anzuzeigen (Three.js oder Babylon.js).

  😊
Wenn Sie nicht wissen, wo Sie anfangen sollen, empfehlen wir Ihnen, einen Blick [hier](https://threejsfundamentals.org/) zu werfen oder einfach dem [Discord-Kanal](https://discord.com/invite/g7Uzn2KSwB) beizutreten und Hallo zu sagen.

## Warum ist IFC.js anders?

IFC.js ist eine Bibliothek, die von und für JavaScript geschrieben wurde, eine der wohl am weitesten verbreiteten Sprachen. Das bedeutet, dass sie mit Webbrowsern, Desktop- und mobilen Anwendungen kompatibel ist. JavaScript ist außerdem eine der am einfachsten zu erlernenden Sprachen und ermöglicht die Erstellung von Benutzeroberflächen mit HTML und CSS. Mit anderen Worten: Das Erstellen von BIM-Anwendungen mit IFC.js ist so einfach wie das Erstellen einer Webseite.

### Geschwindigkeit

  ⚡️
Wenn Sie Erfahrung mit Webanwendungen haben, denken Sie vielleicht, dass der Schwachpunkt einer JavaScript-basierten IFC-Bibliothek die Leistung ist. Der Kern der Bibliothek ist jedoch in C in Kombination mit WebAssembly und [Emscripten](https://emscripten.org/) entwickelt. Sie ist speziell auf **maximale Leistung** ausgelegt, sodass IFC.js sich der Geschwindigkeit und Leistung von Desktop-Anwendungen annähert, während es direkt im Browser läuft.
